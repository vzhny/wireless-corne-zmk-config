nice!view Pixel-Perfect Mockup

Direct 1:1 recreation of what config/widgets/blecorne_central.c / blecorne_peripheral.c draw onto the physical 68×160px nice!view buffer, overlaid at true scale on the actual module photo (niceview_pinout.png). Reflects git HEAD exactly: status strip renders at the physical top, layer name at the physical bottom (no layer circles - removed, they were too small to read reliably). Text and icons render with the actual embedded fonts (PixelOperatorMono.ttf, JetBrainsMonoNLNerdFont-Regular.ttf) converted into config/widgets/fonts/*.c via lv_font_conv - not an approximation, the same glyph shapes the firmware itself uses (rendered here by the browser's own font engine rather than LVGL's, so anti-aliasing/hinting can differ very slightly at the pixel level, but the glyphs are the real ones).

Pixel mapping. Derived directly from LVGL's actual rotate270_l8 source plus this project's widget offsets: each of the three 68×68 canvases (status/mods/layer) maps to the final 68w×160h physical buffer via physical_col = x, physical_row = y + BASE, where BASE is a fixed function of the widget's x offset (BASE = 92 - x for this project's rotation/geometry) - x=-44 → BASE=136 (only 24px of framebuffer remain there before the 160px buffer ends, so only rows 0-23 of that canvas are visible, rest silently clipped), x=24 → BASE=68 (modifier canvas, middle, full 68px available), x=92 (TOP_RIGHT) → BASE=0 (full 68px available, physical top). Status now uses the x=92/top slot (only needs its content in the first 24 rows, rest stays blank/background - harmless); layer name uses the x=-44/bottom slot, which really is hardware-truncated to 24 rows, hence its text sitting at y=5 instead of vertically centered. This closed form already accounts for both the per-canvas rotate_canvas() 270° rotation and the display's own final 270° rotation - no separate rotation simulation needed.
